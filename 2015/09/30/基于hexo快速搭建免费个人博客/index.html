#基于hexo快速搭建免费个人博客

为什么要搭建个人博客

  • 写博客/输出能更好的学
  • 积累，避免实用的知识如过眼云烟般流逝
  • 大部分人的工作都非常单一，容易越来越狭窄，主动给自己加维，增大可能性
    • 你们懂的，博主是苦逼的程序猿
  • 树立个人品牌
  • 当然，同时也可申请微信公众号 ~_~

搭建个人博客的选择

  • 基于wordpress搭建
    • wordpress基于php
  • 基于jekyll搭建
    • jekyll基于ruby
  • 基于hexo搭建
    • hexo基于nodejs
    • 使用github，避免服务器成本
    • 使用自定义域名，个性化，github毕竟太专业 -_-
  • 为什么选用hexo
    • hexo搭建比较简单
    • hexo支持markdown
    • 就hexo了，下面，咱们开始，go

hexo是什么

  • hexo是基于nodejs的静态博客框架
  • 注意，上面这句话表达的意思是
    • hexo是用nodejs写的
    • hexo是静态博客框架，即不包括数据库等复杂的动态操作
  • 我们需要动态功能吗？哪些地方需要呢？不急，后面有讲

安装hexo

  • 安装git和brew，mac默认已安装，本文不做介绍，若未安装，请google

  • 安装nodejs 

    brew install node

  • 因npm的源在国外，下载第三方包非常慢，特修改node源为国内源，如下 

    vim ~/.npmrc   //打开配置文件     
registry =https://registry.npm.taobao.org   //写入配置文件

  • 安装hexo

    sudo npm install -g hexo

  • 建立工作目录

    cd ~/Documents
mkdir hexo
cd hexo
hexo init

  • bash提示

    [info] Copying data
[info] You are almost done! Don't forget to run `npm install` before start blogging with Hexo!

  • 根据提示操作

    npm install

  • 安装node插件

    npm install hexo-generator-index --save
npm install hexo-generator-archive --save
npm install hexo-generator-category --save
npm install hexo-generator-tag --save
npm install hexo-server --save
npm install hexo-deployer-git --save

  • 启动hexo,访问

    url: http://iopenthink.com # 保证评论转发到微博第三方时自带的链接正常

  • 通过修改后的_config.xml重新生成静态文件

    hexo g
hexo d

  • 访问你的域名，恭喜，再次成功，中央电视台发来贺电 ~_~

总结

  • 安装hexo相对来说还算比较简单，大约2小时可以把上述流程跑起来
  • 使用域名映射时，一定要注意CNAME文件的路径，否则，域名映射不会生成
  • 配置多说时，一定要注意_config.xml文件的路径，不要修改到了根路径下的_config.xml，否则不会生效
  • 修改完配置，或写完文章，一定要执行hexo g;hexo d，否则不会生效
